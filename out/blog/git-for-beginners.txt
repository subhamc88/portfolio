<h2>Introduction to VCS (Version Control System)</h2>
<h3>What is a VCS ?</h3>
<p>It is a tool for tracking and managing changes in code and files of the software.
Enabling you to navigate through the various changes to code of your software.
This helps developers to contribute and collaborate on a project without conflicts.</p>
<h3>Benefits of VCS</h3>
<ul>
<li><h4>Change History</h4>
</li>
</ul>
<p>Provides a detailed history of changes to the codebase.
Including what was changecd, who made the change and why it was made.</p>
<ul>
<li><h4>Improved Collaboration</h4>
</li>
</ul>
<p>Developers can work on different parts of the project simultaneously without interfering in other&#39;s work.</p>
<ul>
<li><h4>Disaster Management</h4>
</li>
</ul>
<p>Ability to recover a previous version, reduces the risk of loosing work
In simple words a backup image of your project at every change.</p>
<ul>
<li><h4>Code Quality</h4>
</li>
</ul>
<p>Code reviews and continuous integration improves the overall quality of the codebase.</p>
<ul>
<li><h4>Experimentation</h4>
</li>
</ul>
<p>Developers can create branches to experiment with new features or fixes without effecting the main codebase.
Once the desired changes are verified, the new branch can be merged to the main codebase.</p>
<h3>Different Types of VCS</h3>
<ul>
<li><h4>Local Version Control System</h4>
</li>
</ul>
<p>All the changes done to the project are stored on the local machine.</p>
<ul>
<li><h4>Centralized Version Control System (CVCS)</h4>
</li>
</ul>
<p>All the changes done to the project are stored on a central server.</p>
<ul>
<li><h4>Distributed Version Control System (DVCS)</h4>
</li>
</ul>
<p>Each developer holds a full copy of the project change history.</p>
<h2>Introduction to Git</h2>
<h3>History</h3>
<p>Git was created by Linux Torvalds in 2005 for the development of linux kernel.
(He is the inventor of linux)
It is a free and open-source DVCS (Distrubuted Version Control System).
It was designed to handle small to very large projects with speed and efficiency.</p>
<h3>How Git is different from other VCS ?</h3>
<p>Git is a DVCS, that means each developer has the full change history of the project.
This allows for faster and efficient operations and more flexible workflows.</p>
<h3>Key Freatures of Git</h3>
<ul>
<li><h4>Branching and Merging</h4>
</li>
</ul>
<p>Working on multiple features and fixes by multiple developers simultaneously.</p>
<ul>
<li><h4>Distributed Development</h4>
</li>
</ul>
<p>Each developer has a copy of the main codebase.</p>
<ul>
<li><h4>Speed</h4>
</li>
</ul>
<p>Fast and performant for larger projects.</p>
<h2>Installing Git</h2>
<h3>Windows</h3>
<ol>
<li><p>Download Git from the <a href="https://git-scm.com/download/win">official website</a>.</p>
</li>
<li><p>Run the installer and follow the prompts.</p>
</li>
<li><p>Verify installation by opening Command Prompt and typing:</p>
<pre><code class="language-git">git --version
</code></pre>
</li>
</ol>
<p>and press enter.</p>
<h3>MacOS</h3>
<ol>
<li><p>Open Terminal and install Homebrew if not already installed:</p>
<pre><code class="language-bash">/bin/bash -c &quot;$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)&quot;
</code></pre>
</li>
<li><p>Install Git:</p>
<pre><code class="language-git">brew install git
</code></pre>
</li>
<li><p>Verfy installation by typing :</p>
<pre><code class="language-git">git --version
</code></pre>
</li>
</ol>
<p>and press enter.</p>
<h3>Linux</h3>
<ol>
<li><p>Open Terminal and use your package manager to install git.</p>
</li>
<li><p>Verfy installation by typing :</p>
<pre><code class="language-git">git --version
</code></pre>
</li>
</ol>
<p>and press enter.</p>
